### 1. C++29 最大亮点已可查阅？—— 全面列出所有未定义行为的 Annex [🔗](https://www.meetingcpp.com/blog/items/One-of-the-best-Cpp29-features-is-already-available-.html)
> **来源**：Meeting C++

- **一句话总结**：C++29 将首次把一份完整的"未定义行为（UB）与 IFNDR 清单"纳入标准 Annex，委员会同步推进 P3100R6 对其进行系统性修复。
- **核心要点**：
  - 提案 **P3596** 历经多年推进（从 2019 年的 P1705R1 开始），目标是列出 C++ 中所有已知的显式 UB 和 IFNDR（ill-formed, no diagnostic required），并为每项赋予官方命名
  - IFNDR 特指"本该编译失败却在某些编译器上通过"的程序，是最隐蔽的跨平台陷阱之一
  - 配套提案 **P3100R6** 负责为清单中每一项寻求解决方案——第一阶段聚焦**运行时 UB**（编译期/链接期 UB 和 IFNDR 后续处理）
  - 意义：这是委员会首次系统性承诺降低 C++ 的"未定义行为面积"，是 C++ 迈向更安全语言的关键信号；未来多个标准版本将持续推进修复

## 📜 C++11实用技巧

### 2. maybe_ptr：用 range-for 语法优雅处理可空指针 [🔗](https://lzon.ca/posts/tips/cpp-maybe-type/)
> **来源**：lzon.ca

- **一句话总结**：一种利用 range-based for 循环语法糖实现空指针安全访问的 `maybe_ptr<T>` 小工具，代码简洁（~60 行），灵感来自 C++26 的 `std::optional` range-for 提案。
- **核心要点**：
  - 核心思路：将可空指针包装为"零或一个元素的迭代器范围"，指针非空时 range-for 执行一次迭代，为空时零次迭代
  - 实现包含 `maybe_iterator<T>`（标为 `random_access_iterator_tag`）和 `maybe_ptr<T>`（提供 `begin()/end()`），`operator++` 直接将迭代器置为 end
  - 使用方式：`for (auto&& val : maybe_ptr<T>(ptr)) { /* 指针非空才执行 */ }`，比传统 `if (ptr) { ... }` 更简洁且意图表达更清晰
  - 生产建议：适合短生命周期的局部指针检查场景，但滥用迭代器标签（`random_access_iterator_tag`）可能让 STL 算法意外展开，谨慎使用

## 🛠 工具与库

### 3. YaFF：Yandex 开源高性能零拷贝 Protobuf 序列化库 [🔗](https://github.com/yandex/yaff)
> **来源**：GitHub (Yandex)

- **一句话总结**：YaFF（Yet Another Flat Format）是 Yandex 新开源的 C++ 序列化库，提供兼容 Protobuf schema 的零拷贝线格式，读取性能接近原生 C++ struct。
- **核心要点**：
  - **零拷贝**：内存映射（mmap）即可直接访问，无需解析步骤，消除序列化/反序列化开销
  - **自适应布局**（Adaptive Layouts）：支持显式配置或运行时调整，灵活权衡速度、内存开销与 schema 灵活性
  - **Protobuf 原生兼容**：直接使用 `.proto` 文件作为唯一 schema 源，尊重 Protobuf 语义；支持双向消息转换（YaFF ↔ Protobuf），可逐模块渐进式迁移
  - 使用方式：`yaff::Serialize<ProtoType>(msg)` 得到 buffer，`yaff::ReadMessage<ProtoType>(data)` 直接零拷贝访问字段
  - **规划中**：列式布局（优化分析/ML 管道）、自动化自适应布局、多语言绑定
  - Apache 2.0 许可，支持 CMake/Conan 集成
