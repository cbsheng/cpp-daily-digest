### 1. C++26: constexpr Virtual Inheritance [🔗](https://www.sandordargo.com/blog/2026/07/01/cpp26-constexpr-virtual-inheritance)
> **来源**：Hacker News / Sandor Dargo's Blog

- **一句话总结**：C++26 通过 P3533R2 提案移除了 `constexpr` 对虚继承的禁令，这是 std::iostream 被 constexpr-ification 的最后一道门槛。
- **核心要点**：
  - C++20 已允许 `constexpr virtual` 函数，但禁止包含虚基类的类型的构造/析构为 constexpr；Clang 更是拒绝了所有成员函数
  - P3533R2（Hana Dusíková）移除了这一限制：含虚基类的类型现在可在编译期构造，成员函数也可 constexpr 使用
  - **实战意义**：`std::ios_base` 是整个 iostream 层次的基础，它使用虚继承——因此之前无法 constexpr。解除后，`std::chrono` 异常类型、`basic_istream_view` 等编译期解析都将被解锁
  - GCC 16 已在投票后一周落地实现（Jakub Jelinek）
  - 更深层含义：`constexpr` 正从「被一批语法限制守卫的关键字」收敛为「仅由 [expr.const] 中求值属性决定」的简单 opt-in 模型。虚继承放开后，唯一剩余限制只剩协程（P3367 正在处理，目标 C++29）

### 2. memkit — 面向 MCU/MPU 的嵌入式 C++ 容器库 [🔗](https://github.com/jameslavrenz/memkit)
> **来源**：GitHub (38 stars)

- **一句话总结**：专为裸机和 RTOS 设计的零开销容器库，C++/C 双 API，32+ 种容器，无异常、无 RTTI、无虚拟派发。
- **核心要点**：
  - **架构特色**：C++ templates → detail/*_core<Policy> → C API wrappers，同一算法实现供两套 API 共用，杜绝逻辑重复
  - **零开销承诺**：无内部互斥锁、无虚函数、无 RTTI，多态全部编译期完成（template + policy struct），`-fno-exceptions -fno-rtti` 项目内强制
  - **两大容器类别**：
    - **无锁三件套**（C++ only）：`SpscQueue<T>`（等待自由 SPSC）、`MpscQueue<T>`（无锁 MPSC，Vyukov 风格）、`DoubleBuffer<T>`（等待自由乒乓缓冲区）——适合 ISR ↔ 主循环通信
    - **单线程容器**（裸机优化）：`HashMap`、`Vector`、`BTree`、`FlatMap`、`EnumMap`、`Ring`、`Queue`、`ObjPool`、`HandlePool`、`IntrusiveList`、`TimerWheel`、`SparseSet` 等
  - **内存模型**：支持静态缓冲区、Arena（bump allocator）、堆（仅 MPU）、mmap（仅 MPU）四种模式，MCU 零堆分配
  - **实用工具**：`SmallString<N>`、`SmallBuffer<N>`、`FixedVariant<Ts...>`、`TokenBucket`（速率限制）、`BitReader/BitWriter`、`MovingAverage`、`LookupTable`（校准表）、`RingLog`（飞行记录器）
  - **构建**：`make all` 即用，CMCake 支持，CI 覆盖 Clang 21 + GCC 14 + ASan/UBSan + macOS
  - **API 完备度 v0.2**: C++ 32 个容器 + C API 14 个容器（8 个 tier-1 MCU 可用，6 个 tier-2 仅 MPU）+ 测试/例子完善
