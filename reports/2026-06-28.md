### 1. C++26: string and string_view improvements [🔗](https://isocpp.org//blog/2026/06/cpp26-string-and-string-view-improvements-sandor-dargo)
> **来源**：isocpp.org (Sandor Dargo)

- **一句话总结**：C++26 通过 P2495R3 为 stringstreams 新增接受 `string_view` 的构造函数，同时简化了 string 与 string_view 的拼接操作。
- **核心要点**：
  - `std::string_view`（C++17）提供对字符串的只读访问，可替代 `const string&` 参数，性能显著提升
  - C++23 虽然引入了 spanstreams，但 stringstreams 因语义差异仍被广泛使用，却一直缺少 string_view 构造函数
  - P2495R3 解决了这个问题：过去要从 `string_view` 构造 stringstream 需手动转换为 `string`，现在可以直接传入
  - 纯库层新增改动，不会破坏已有代码
  - 该更改已在 Clang 19 中可用

- **核心要点**：
  - Ranges 提供懒惰的按需计算，改善 I/O 效率和可组合性；并行算法利用多任务加速计算密集型负载
  - 在实际中，许多 Range 操作（尤其是非随机访问源）由于懒惰的 pull-based、一次一个元素的特性，本质上是顺序的
  - 演讲演示了如何构建懒惰、可组合且并行友好的 Range Pipeline，高效表达 filter、scan、flatten 等操作
  - 通过真实案例展示这些技术如何在最小编程开销下实现强并行加速
  - 适合库设计师、性能优化爱好者和想让 Range 代码更好扩展的开发者

- **核心要点**：
  - 纯 C++ 实现，基于 ggml，无 Python 运行时依赖；支持 CUDA/Vulkan/Metal 后端
  - 已支持 20+ 模型家族：Chatterbox（TTS/语音克隆）、Qwen3-TTS/ASR/ForcedAligner、Silero VAD、SeedVC 语音转换、VeVo2 音乐/歌声生成等

  - 支持量化（q8_0 等），但需逐模型验证质量与稳定性
