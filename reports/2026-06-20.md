## ⚡ 性能优化

### 1. 操作耗时速查图（第二版）：现代 CPU 时钟周期大全 [🔗](https://6it.dev/blog/infographics-operation-costs-in-cpu-clock-cycles-take-2-80736)
> **来源**：Hacker News

- **一句话总结**：《Efficient C++ Programming for Modern 64-bit CPUs》第 4 章第二部分，给出 Skylake-X / Zen 2~4 / Cortex-A78 / Alder Lake 等现代 CPU 各操作的实际时钟周期数。
- **核心要点**：
  - **整数乘除**：MUL 32/64 位差异不大（3 周期），但 **DIV 仍是性能杀手**——64 位除法在不同架构上可达 9~96 周期（Zen 4 约 9-19 周期，Skylake-X 高达 37-96 周期）
  - **RTTI**：`dynamic_cast<>` 比虚函数调用贵 **5 倍**；但 RTTI 不会增加对象大小（复用 vfptr），只增加代码体积
  - **C++ 异常 vs 返回值**：零成本异常在无异常时确实零开销；异常发生时约 2700~5000 周期。经验法则：每 1 万次调用少于 1 次异常，异常更优
  - **原子操作/CAS**：同一 NUMA 节点约 15 周期；多 socket 场景可达 **300-600 周期**。原子操作还会阻止指令级并行，使带宽下降最多 30 倍
  - **函数调用开销**：直接调用约 15-30 周期（参数少时）；间接调用（函数指针）20-50 周期；虚函数 30-60 周期
  - **内联的隐藏收益**：不仅省去调用开销，更重要的是使编译器能跨函数边界优化（如 `square(x) + cube(x)` 内联后可合并为 `(x*x)*(1+x)`，节省一次乘法）
  - **线程局部存储（TLS）**：GCC/Clang 下 1 次额外间接寻址，MSVC 下最多 **3 次间接寻址**
  - **线程上下文切换**：直接切换约 2000 周期，但缓存失效导致的间接开销可达 **1~300 万周期**（严重时 3000 万周期）。这也解释了 Windows CRITICAL_SECTION 默认自旋 4000 次的合理性

## 🏗 新特性实战

### 2. P3100R6：系统性消除 C++ 标准未定义行为的整体框架 [🔗](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2026/p3100r6.pdf)
> **来源**：Hacker News（WG21 论文，已获 EWG 强共识，目标 C++29）

- **一句话总结**：Timur Doumler & Joshua Berne 提出首个系统性处理 C++ 核心语言 80 种 UB 的完整策略，包含完整标准措辞（98 页），已通过 SG21/SG23/EWG 三轮审查。
- **核心要点**：
  - **穷举枚举**：人工审查现行 C++ 工作草案，识别出 **80 个显式核心语言 UB**（合并了 Shafik Yaghmour 的独立清单），每个都有稳定 ID
  - **十大分类**：初始化(1) / 边界(5) / 类型与生命周期(50) / 算术(9) / 线程(2) / 顺序(1) / 假设(1) / 控制流(5) / 替换函数(3) / 协程(3)
  - **内存安全占比 70%**：初始化 + 边界 + 类型与生命周期共 56 个 UB，占全部 80 个的 **70%**
  - **三大诊断发现**：
    - 仅 **23.75%（19 个）** 可在运行时局部诊断（如除零、null 解引用）
    - **72.5%（58 个）** 需要全局插桩（如指针生命周期追踪）
    - **3.75%（3 个）** 根本不可诊断（如 `[[assume]]` 假）
  - **七种治理工具**：特性删除、行为精化（refined behaviour）、错误行为（erroneous behaviour）、运行时检查、语言子集、注解、新特性
  - **核心方案**：基于 C++26 已采纳的 **Contracts（P2900R14）**，对运行时可检查的 UB 自动插入隐式契约断言（implicit contract assertions）。检查失败后程序**不 UB，而是进入错误行为（erroneous behaviour）**，继续执行，给用户完全的控制权
  - **已获关键投票**：SG21 一致支持 → EWG "强共识"（39 SF / 19 F）→ SG23 支持纳入 C++29
  - **附录 A**：80 个 UB 的完整清单（与 P3596R2 的 UB/IFNDR 附录同步），每个标注了类别、可诊断性、是否存在替代行为
