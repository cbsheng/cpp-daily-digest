### 1. 用 C++26 静态反射在编译期解析 JSON [🔗](https://lemire.me/blog/2026/06/14/parsing-json-at-compile-time-with-c26-static-reflection/)
> **来源**：Daniel Lemire's Blog (via Hacker News)

- **一句话总结**：Daniel Lemire 展示了如何用 C++26 的 `#embed` + 静态反射 + simdjson，在编译期直接将 JSON 配置文件解析为类型安全的 C++ 对象，零运行时开销。
- **核心要点**：
  - C++26 的 `#embed` 指令将文件内容在编译期嵌入为字节数组，配合 `constexpr` 让编译器在常量求值期就可访问
  - simdjson 新增 `simdjson::compile_time::parse_json<json_data>()` 接口，基于静态反射从 JSON 键名自动合成 struct 类型——无需手写解析代码
  - 编译期解析的结果可用 `static_assert` 验证内容，JSON 损坏直接导致编译失败，零运行时异常
  - 任何基于该常量的计算（如 `width * height`）在 `-O3` 下被完全折叠为立即数，JSON 从二进制中彻底消失
  - 当前需 GCC 16 + `-std=c++26 -freflection` + `#define SIMDJSON_STATIC_REFLECTION 1`

### 2. locate-anything.cpp：NVIDIA 视觉定位模型的 C++/ggml 移植 [🔗](https://github.com/mudler/locate-anything.cpp)
> **来源**：GitHub (1275 ⭐)

- **一句话总结**：来自 LocalAI 团队，将 NVIDIA LocateAnything-3B（开放词汇目标检测 VLM）移植到 C++17 + ggml，纯 CPU 推理无需 Python 运行时，比官方 PyTorch 快 1.7-3 倍。
- **核心要点**：
  - 模型架构：Qwen2.5-3B (LM) + MoonViT (视觉) + 2层 MLP 投影器；检测在 token 空间完成，输出 `<0>`-`<1000>` 坐标 token
  - 检测结果与官方实现**逐框一致**（IoU 1.000），已验证通过
  - CPU 性能（Ryzen 9 9950X3D）：混合模式 22.32s vs 官方 69.06s（**3.09×**）；q8_0 量化后低至 4.89s（**4.8×**）
  - 支持量化（q8_0/q6_k/q5_k/q4_k），q8_0 仅 6.3GB，框级无损
  - 构建：CMake，可选 CUDA/Metal/Vulkan 后端，ggml 作为 submodule 内嵌
  - 命令行使用：`locate-anything-cli detect --model model.gguf --input img.jpg --prompt "Locate: person" --annotated out.png`
