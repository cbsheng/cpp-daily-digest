### 1. C++26: Cleaning up string literals [🔗](https://www.sandordargo.com/blog/2026/06/10/cpp26-string-literals-cleaned-up)
> **来源**：Sandor Dargo's Blog (Hacker News)

- **一句话总结**：C++26 通过两份提案（P2361R6 + P1854R4）正式规范了字符串字面量的行为——未求值字符串禁止编码前缀，求值字符串中不可编码字符改为编译错误。
- **核心要点**：
  - **P2361R6（未求值字符串）**：`static_assert`、`_Pragma`、`#line`、`[[nodiscard]]` / `[[deprecated]]`、`extern` 链接、`asm` 中出现的字符串不再允许编码前缀（`L`、`u8`、`u`、`U`），且禁止数字转义序列（`\x1B`、`\077`），只允许通用字符名和简单转义序列（`\n`、`\t`）。编译器保留原始字符序列用于诊断。
  - **P1854R4（求值字符串 + 多字符字面量）**：若字符串中某字符无法在关联编码中表示，行为从「实现定义」（MSVC 静默替换为 `?`）改为**编译错误**。需要 Unicode 字符串请用 `u8` 前缀 + `char8_t`。多字符字面量（如 `'abcd'`）中的每个元素必须可表示为单个编码单元——组合字符（如 e + 重音符号）会被编译器捕获。
  - **向后兼容**：对 9000 万行开源代码的调查显示，几乎没有实际代码受此影响，仅 Clang 自身测试套件中有使用编码前缀的 `_Pragma`/`static_assert`。
  - P1854R4 作为 defect report 追溯至 C++98，编译器需立即生效。

### 2. Fast Branchless Quicksort Using Sorting-Networks with C and C++ Interface [🔗](https://tiki.li/blog/blqsort)
> **来源**：Hacker News

- **一句话总结**：利用排序网络（sorting networks）实现无分支快速排序，单线程版比 `std::sort` 快 30%~64%，多线程版快 2~4x。
- **核心要点**：
  - **核心思想**：将小数组排序时通常由 CPU 分支预测失败造成的惩罚，替换为无分支的排序网络（使用条件移动指令 CMOV），大幅提升流水线效率。
  - **基准测试（1 亿个 int）**：
    | 实现 | Apple M1 | AMD Ryzen |
    |------|----------|-----------|
    | std::sort | 1.33s | 5.56s |
    | pdqsort | 1.33s | 2.81s |
    | **blqsort（单线程）** | **0.93s** | **1.98s** |
  - 多线程版本在 M1 上仅 **0.90s**（vs `std::sort` 3.46s，pdqsort 3.46s）。
  - 提供纯头文件库：`blqs.h`（C++ 单线程）、`blqs_thr.h`（C++ 多线程）、`blqsort.h`（C 单线程）、`blqsort_thr.h`（C 多线程）。
  - 支持自定义数据类型，C 接口通过函数指针，C++ 接口通过模板。
  - **实战建议**：对大规模数据排序（数据库、数据分析、科学计算）有显著收益，小数据集效果不明显。
