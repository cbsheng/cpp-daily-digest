### 1. GCC C/C++ Vector Extensions：无需平台内联汇编的便携 SIMD 编程 [🔗](https://gcc.gnu.org/onlinedocs/gcc/Vector-Extensions.html)
> **来源**：Hacker News

- **一句话总结**：GCC 内置的向量扩展让开发者用纯 C/C++ 语法写 SIMD 代码，无需 x86 SSE/AVX 或 ARM NEON 等平台特定内联函数。
- **核心要点**：
  - 核心语法：`typedef int v4si __attribute__((vector_size(16)));` — 声明 4 个 int 组成的 16 字节向量
  - 支持基本整数类型（signed/unsigned），`vector_size` 需为 2 的幂
  - 支持标准 C 运算符做逐元素运算：`+`、`-`、`*`、`/`、`unary minus`、`~`、`|`、`&`、`^` 等
  - 标量广播自动扩展：`a = b + 1` 等价于 `a = b + {1,1,1,1}`
  - 支持向量比较运算符，返回符号掩码
  - 可搭配 union+intrinsic 混合使用：`x.mm = _mm_adds_epu8(x.mm, y.mm)` 同时在 union 内操作普通字段和向量字段
  - **最大优势**：一套代码兼容 x86/ARM/RISC-V，编译器自行映射到目标平台 SIMD 指令

### 3. Fail-safe Application Logger：C/C++ 日志生成防崩溃 API [🔗](https://github.com/wandi-ssal/fail-safe-application-logger)
> **来源**：Hacker News

- **一句话总结**：一个在运行时对 printf 风格日志做 18 种边界参数验证的 C API 库，阻止因 NULL 指针、类型不匹配等导致的日志级崩溃。
- **核心要点**：
  - 自动解码日志消息字符串和参数，逐元素/值对做运行时分析
  - 18 种 Bound Parameter（边界参数）验证技术（含 PDF 文档和示例）
  - 检测内容包括：NULL 指针传入 `%s`、0 或负数作为格式参数、类型不匹配、恶意 printf 攻击模式
  - 验证失败时返回错误码并跳过该日志，**不中断应用正常运行**
  - 提供 C++ 兼容的 C 接口：`LoggerStartUp()`、`LoggerAppsMsg()`、`LoggerShutDown()`
  - 已发布 Linux arm64 二进制库，含 GPG 签名验证
  - 适合嵌入式、高可靠性场景——日志不能成为系统崩溃的入口
